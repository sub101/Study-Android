### 1. 패키지

: 클래스와 인터페이스가 많아지면 관리하기가 어렵기 때문에 패키지 단위로 묶어서 관리

- 사용자가 생성한 클래스가 포함될 패키지는 *.kt 파일 첫행에 
  'package 패키지명' 으로 지정

### 2. 제네릭스

: 데이터 형식의 안전성을 보장하는 데 사용

- 제네릭스를 사용해 strList에 문자열만 들어가도록 설정 가능
- 코드의 마지막 행은 컴파일 오류가 발생하기 때문에 미리 문제점을 확인 가능
